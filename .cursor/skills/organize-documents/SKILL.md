# 文档整理技能

## 核心原则

1. **保留原意**：不添加用户没有表达的内容，只做结构优化
2. **拆分独立**：每个文件只包含一个独立主题
3. **命名清晰**：文件名能准确反映内容
4. **格式统一**：使用一致的标题层级和结构

## 整理流程

### 第一步：分析现有文件

1. 列出文件夹中所有文件
2. 读取每个文件的内容
3. 识别以下问题：
   - 文件命名过于笼统（如"想法.md"、"问题.md"）
   - 单个文件包含多个不相关主题
   - 空文件
   - 拼写/用词错误

### 第二步：制定整理方案

向用户展示：
- 当前问题分析
- 建议的新文件结构
- 各文件的改动说明

等待用户确认后再执行。

### 第三步：执行整理

1. 创建新文件（重命名或拆分后的）
2. 删除旧文件
3. 验证最终结构

## 文件命名规范

### 命名模式

```
[分类前缀]-[具体描述].md
```

### 常用分类前缀

| 前缀 | 适用场景 |
|------|----------|
| `项目想法-` | 项目构思、产品想法 |
| `技术问题-` | 技术疑问、待研究问题 |
| `故障排查-` | 问题诊断、解决方案 |
| `学习计划-` | 学习路径、计划安排 |
| `工具使用-` | 工具教程、使用技巧 |

### 命名示例

| 原名 | 改后 |
|------|------|
| 想法.md | 项目想法-OpenClaw应用.md |
| 问题.md | 技术问题-响应式分辨率实现.md |
| AI学习路径.md | AI学习路径-三月计划.md |

## 内容整理规范

### 拆分原则

当一个文件包含多个不相关主题时，拆分成独立文件：

**拆分前**：
```
想法.md
├── OpenClaw应用想法
└── 账单管理系统想法（不相关）
```

**拆分后**：
```
项目想法-OpenClaw应用.md
项目想法-账单管理系统.md
```

### 内容优化

1. **添加层级标题**：用 `##` 和 `###` 组织内容结构
2. **修正拼写**：统一术语拼写（如 openclaw → OpenClaw）
3. **保留原意**：不添加用户未表达的内容
4. **删除空文件**：移除没有内容的文件

## 输出格式

整理完成后，向用户展示：

```markdown
## 整理结果

[文件夹结构树]

## 主要改动

| 改动类型 | 说明 |
|---------|------|
| 文件重命名 | ... |
| 内容拆分 | ... |
| 格式优化 | ... |
| 拼写修正 | ... |
```
